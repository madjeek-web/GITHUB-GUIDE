## Français

Merci de l'intérêt que vous portez à ce projet. Ce guide de contribution explique comment participer de façon efficace et respectueuse.

### Avant de commencer

- Lisez le [README.md](README.md) pour comprendre l'objectif du projet.
- Vérifiez les [issues existantes](https://github.com/madjeek-web/GITHUB-GUIDE/issues) pour éviter les doublons.
- Si vous avez une idée nouvelle, ouvrez d'abord une issue pour en discuter avant de commencer à coder.

### Comment signaler un bug

1. Vérifiez que le bug n'a pas déjà été signalé dans les issues.
2. Ouvrez une nouvelle issue avec le template "Bug Report".
3. Décrivez le problème de façon précise : ce que vous avez fait, ce que vous avez obtenu, ce que vous attendiez.
4. Ajoutez des captures d'écran si utile.

### Comment proposer une amélioration

1. Ouvrez une issue avec le template "Feature Request".
2. Expliquez clairement le besoin et la valeur ajoutée.
3. Proposez une solution si vous en avez une.

### Processus de contribution par Pull Request

1. Forkez le dépôt sur votre compte GitHub.
2. Clonez votre fork localement.

```bash
git clone git@github.com:votre-utilisateur/GITHUB-GUIDE.git
cd GITHUB-GUIDE
```

3. Créez une branche dédiée à votre contribution.

```bash
git checkout -b feature/ma-contribution
```

4. Faites vos modifications.
5. Vérifiez la syntaxe Markdown de vos fichiers.
6. Commitez vos changements avec un message clair.

```bash
git commit -m "docs: add section on SSH configuration"
```

7. Poussez votre branche.

```bash
git push origin feature/ma-contribution
```

8. Ouvrez une Pull Request sur GitHub depuis votre branche vers `main`.

### Standards de contribution

- Tout contenu ajouté doit être exact, vérifiable et à jour.
- Les exemples de commandes doivent être testés avant d'être soumis.
- Le style rédactionnel doit rester pédagogique, simple et clair.
- Les contributions bilingues (français et anglais) sont encouragées.
- Respectez la structure existante du document.

### Code de conduite

Ce projet respecte un code de conduite simple : soyez respectueux et bienveillant envers tous les contributeurs. Voir [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## English

Thank you for your interest in this project. This contribution guide explains how to participate effectively and respectfully.

### Before you start

- Read [README.md](README.md) to understand the project's goal.
- Check [existing issues](https://github.com/madjeek-web/GITHUB-GUIDE/issues) to avoid duplicates.
- If you have a new idea, open an issue first to discuss it before starting to write.

### How to report a bug

1. Check that the bug has not already been reported in the issues.
2. Open a new issue using the "Bug Report" template.
3. Describe the problem precisely : what you did, what you got, what you expected.
4. Add screenshots if helpful.

### How to propose an improvement

1. Open an issue using the "Feature Request" template.
2. Clearly explain the need and the added value.
3. Propose a solution if you have one.

### Pull Request contribution process

1. Fork the repository to your GitHub account.
2. Clone your fork locally.

```bash
git clone git@github.com:your-username/GITHUB-GUIDE.git
cd GITHUB-GUIDE
```

3. Create a branch dedicated to your contribution.

```bash
git checkout -b feature/my-contribution
```

4. Make your changes.
5. Check the Markdown syntax of your files.
6. Commit your changes with a clear message.

```bash
git commit -m "docs: add section on SSH configuration"
```

7. Push your branch.

```bash
git push origin feature/my-contribution
```

8. Open a Pull Request on GitHub from your branch to `main`.

### Contribution standards

- All added content must be accurate, verifiable, and up to date.
- Command examples must be tested before being submitted.
- The writing style must remain pedagogical, simple, and clear.
- Bilingual contributions (French and English) are encouraged.
- Respect the existing structure of the document.

### Code of conduct

This project follows a simple code of conduct : be respectful and kind to all contributors. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
